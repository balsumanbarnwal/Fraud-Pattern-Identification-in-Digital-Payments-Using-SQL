# Fraud-Pattern-Identification-in-Digital-Payments-Using-SQL
This project focuses on detecting fraudulent activities in digital payment transactions using advanced SQL techniques. The objective was to analyze large-scale transactional data to identify suspicious patterns, unusual customer behavior, and high-risk transactions that may indicate fraud.
This project focuses on detecting fraudulent activities in digital payment transactions using advanced SQL techniques. The objective was to analyze transactional data to identify suspicious patterns, abnormal customer behavior, and high-risk transactions.
Using PostgreSQL, I performed data cleaning, validation, and transformation to ensure data accuracy. I applied advanced SQL concepts such as CTEs, joins, aggregate functions, subqueries, CASE statements, and window functions to detect fraud indicators. Key analysis included identifying unusually high transaction amounts compared to customer averages, detecting multiple rapid transactions within short time intervals, and flagging transactions from unusual locations or devices.
